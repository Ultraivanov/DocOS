# DocOS — MVP Product Spec

## 1. Product summary

DocOS is a workspace for designing, testing, and improving AI behavior through configuration files.

Core loop:
describe → configure → test → improve

---

## 2. Goal of MVP

Validate that users:
- modify configs
- run multiple tests
- see improvement in AI output

---

## 3. Core User Flow

1. User writes config
2. User enters test input
3. User runs test
4. User sees output
5. User edits config
6. User runs again
7. User compares mentally (MVP)

---

## 4. Core Features

- Config editor (textarea)
- Test input
- Run button
- Output panel

---

## 5. What we are testing

We are NOT testing UI.

We are testing:

- does user iterate?
- does output improve?
- does user feel control?

---

## 6. Success Criteria

- user runs ≥ 3 tests
- user edits config between runs
- user perceives improvement

---

## 7. Failure Criteria

- user runs once
- no edits
- no improvement

---

## 8. Key Insight

The product is not about writing configs.

It is about:

> improving AI behavior through iteration
